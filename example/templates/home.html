Was ist Cookie Love?

djangocms-cookie-love ist ein Django CMS Paket für ein vollständig DSGVO-konformes Cookie-Consent-Management. Es bietet einen modernen, anpassbaren Cookie-Banner mit granularer Kontrolle über Cookie-Gruppen, Versionierung und einer vollständigen Audit-Trail-Dokumentation.

Features

🛡️ DSGVO-Konformität

  • Opt-in by Default – keine vorausgewählten optionalen Cookies
  • Granulare Kontrolle pro Cookie-Gruppe
  • Informierte Einwilligung mit Cookie-Details
  • Jederzeit widerrufbar
  • Vollständiger Audit-Trail
  • IP-Adressen werden nur als SHA-256-Hash gespeichert

🎨 Modernes Design

  • Bootstrap 5 kompatibel
  • Responsive & Mobile-First
  • Barrierefreiheit (ARIA, Keyboard, Focus Trap)
  • CSS Custom Properties für einfaches Theming
  • 3 Banner-Positionen: Unten, Oben, Zentriert
  • Smooth Animations & Dark Mode Support

🔄 Versionierung

  • Consent-Versionen mit Snapshot der Konfiguration
  • Automatische Re-Consent-Aufforderung bei Änderungen
  • Vollständige Historie aller Einwilligungen
  • CSV-Export der Consent-Records

⚡ Entwicklerfreundlich

  • Vanilla JavaScript – kein jQuery nötig
  • Öffentliche JS API: CookieLove.openSettings()
  • REST API für Consent-Verwaltung
  • Django Middleware & Context Processor
  • Template Tags für einfache Integration
  • i18n-ready (Deutsch & Englisch)

Installation

1. Paket installieren

pip install djangocms-cookie-love

2. INSTALLED_APPS

INSTALLED_APPS = [
    ...
    "djangocms_cookie_love",
]

3. Middleware hinzufügen

MIDDLEWARE = [
    ...
    "djangocms_cookie_love.middleware.CookieConsentMiddleware",
]

4. Context Processor (optional)

TEMPLATES = [{
    "OPTIONS": {
        "context_processors": [
            ...
            "djangocms_cookie_love.context_processors.cookie_consent",
        ],
    },
}]

5. URLs einbinden

# urls.py
from django.urls import include, path

urlpatterns = [
    ...
    path("cookie-love/", include("djangocms_cookie_love.urls")),
]

6. Migrationen ausführen

python manage.py migrate

Einrichtung

Admin-Konfiguration

Nach der Installation können Sie im Django Admin unter Cookie Love → Cookie Consent Configurations die Konfiguration anlegen:

  1. Titel und Beschreibungstext für den Banner
  2. Links zu Datenschutzerklärung und Impressum
  3. Banner-Position wählen (Unten / Oben / Zentriert)
  4. Button-Beschriftungen anpassen
  5. Cookie-Gruppen anlegen (Notwendige, Analyse, Marketing, etc.)

Cookie-Gruppen

Jede Cookie-Gruppe enthält:

  • Name & Slug – Anzeigename und technischer Bezeichner
  • Beschreibung – Erklärt dem Nutzer wofür die Cookies verwendet werden
  • Erforderlich – Pflicht-Cookies können nicht deaktiviert werden
  • Cookies – Liste der einzelnen Cookies mit Name, Anbieter, Dauer und Zweck

JavaScript API

Die öffentliche CookieLove API steht global zur Verfügung:

// Cookie-Einstellungen öffnen
CookieLove.openSettings();

// Alle Cookies akzeptieren
CookieLove.acceptAll();

// Nur notwendige Cookies
CookieLove.rejectAll();

// Prüfen ob eine Gruppe akzeptiert wurde
if (CookieLove.hasConsent("analytics")) {
    // Analytics laden
}

// Auf Consent-Änderungen reagieren
CookieLove.onConsent(function(groups) {
    console.log("Akzeptierte Gruppen:", groups);
});

// Einwilligung widerrufen
CookieLove.revokeConsent();

Script Blocking

Skripte können automatisch erst nach Einwilligung geladen werden:

<script type="text/plain" data-cl-consent="analytics"
        data-cl-src="https://www.googletagmanager.com/gtag/js">
</script>

REST API

Methode Endpoint Beschreibung
GET /cookie-love/api/config/ Banner-Konfiguration mit Cookie-Gruppen
GET /cookie-love/api/consent/ Aktuellen Consent-Status abfragen
POST /cookie-love/api/consent/ Consent speichern
POST /cookie-love/api/consent/revoke/ Einwilligung widerrufen

Einstellungen

Alle Einstellungen sind optional und werden in settings.py gesetzt:

Setting Default Beschreibung
COOKIE_LOVE_COOKIE_NAME "cookie_love_consent" Name des Consent-Cookies
COOKIE_LOVE_COOKIE_DURATION 365 Gültigkeit in Tagen
COOKIE_LOVE_COOKIE_SECURE True Secure-Flag (HTTPS only)
COOKIE_LOVE_COOKIE_SAMESITE "Lax" SameSite-Attribut
COOKIE_LOVE_IP_SALT "cookie-love" Salt für IP-Hashing

Middleware & Context Processor

Die Middleware setzt auf jedem Request folgende Attribute:

# In Views oder Templates verfügbar:
request.cookie_consent          # UserConsent-Objekt oder None
request.cookie_consent_required # True wenn Banner angezeigt werden soll
request.cookie_consent_groups   # Liste akzeptierter Gruppen-Slugs

# Im Template via Context Processor:
{% templatetag openblock %} if cookie_consent_required {% templatetag closeblock %}
    {# Banner anzeigen #}
{% templatetag openblock %} endif {% templatetag closeblock %}

{% templatetag openblock %} if "analytics" in cookie_consent_groups {% templatetag closeblock %}
    {# Analytics-Code laden #}
{% templatetag openblock %} endif {% templatetag closeblock %}
